# Windpipe

Go-based automation pipeline for 3D print preprocessing.

This project is being migrated from the previous Windmill-based automation setup to a simpler Go/Restate-oriented system. The current minimal app exposes a local Go HTTP server and preserves the STL preview rendering script in `render_stl.sh`.

Concise flow:

1. backend sends request -> uploads to blob storage 
2. pipeline receives request + key to blob storage 
3. pipeline pulls from blob storage 
4. run printability tests
5. extract image for NSFW AI model (deferred)
6. run slicer 
7. call 3DQue API for handoff 

## Architecture/Structure 

```
backend -> Restate server (journaling lives here) -> Go web server (durable) -> printers
```

## Setup

To set things up, you should have:

- Docker and Docker Compose
- Go
- curl (curl.exe on Windows)
- Recommended: Restate CLI

We will be using Docker for the Restate server during development and running the Go backend on localhost:9080.

## Setup

### 1. Download Go dependencies

cd into this repo and run: 
```bash
go mod download
```

If dependencies have changed, clean up the module file with the following. Do not run this command currently in the repo, which does not contain any restate code being used. This will result in restate being removed from the go.mod file and we will have to reinstall with `go get github.com/restatedev/sdk-go`:

```bash
go mod tidy
```

### 2. Make scripts executable

```bash
chmod +x ./render_stl.sh
```

### 3. Start the Restate dev container

```bash
docker compose up restate
```

The Restate UI is available at:

```text
http://localhost:9070
```

### 4. Start the Go dev server

In a separate terminal:

```bash
go run ./cmd/service
```

By default, the Go server listens on:

```text
http://localhost:9080
```

To use a different port:

```bash
PORT=3000 go run ./cmd/service
```

### 5. Test the local endpoints

```bash
curl http://localhost:9080/
curl http://localhost:9080/healthz
```

## Port Debugging/killing the port when done with dev server 

### macOS / Linux

Check whether port `9080` is active:

```bash
lsof -i :9080
```

Example output includes a process ID under the `PID` column. Stop that process with:

```bash
kill <PID>
```

If the process does not stop:

```bash
kill -9 <PID>
```

### Windows PowerShell

Check whether port `9080` is active:

```powershell
Get-NetTCPConnection -LocalPort 9080
```

Look for the `OwningProcess` value, then stop it:

```powershell
Stop-Process -Id <PID>
```

### Windows Command Prompt

Check whether port `9080` is active:

```cmd
netstat -ano | findstr :9080
```

Stop the process:

```cmd
taskkill /PID <PID> /F
```

## Docker Commands

Start the Restate container:

```bash
docker compose up restate
```

Start all configured containers:

```bash
docker compose up
```

Rebuild and start containers:

```bash
docker compose up --build
```

Stop containers without deleting volumes:

```bash
docker compose down
```

Start with fresh containers while keeping volumes:

```bash
docker compose down
docker compose up --build --force-recreate
```

Reset containers and delete Compose-managed volumes:

```bash
docker compose down -v
```

Use `docker compose down -v` carefully. It deletes local Compose-managed volumes and should be treated as a full local reset.
